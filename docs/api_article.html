View Article via JSON-RPC

Use the get_show_article method to retrieve any published message or article from the ANTD blockchain by providing the transaction ID.

Example cURL Request

curl -X POST http://127.0.0.1:14041/json_rpc \
  -H "Content-Type: application/json" \
  -d '{
    "jsonrpc": "2.0",
    "id": "0",
    "method": "get_show_article",
    "params": {
      "txid": "20805f943cdfefc3925597939c0b2d3f1db082992a9f524af56094d383715e48"
    }
}'

Example Response

{
  "id": "0",
  "jsonrpc": "2.0",
  "result": {
    "article_found": true,
    "block_height": 100,
    "content": "Antd is blockchain with a difference",
    "content_hash": "6a3da855dfc655e6e3dbcf2c2f5545d396508bae6c7719c22fedc72cc7350022",
    "in_pool": false,
    "publisher": "AntD Developer",
    "status": "OK",
    "title": "ANTD STARTS",
    "tx_extra_nonce_hex": "415254430b414e5444205354415254530024416e746420697320626c6f636b636861696e2077697468206120646966666572656e63650e416e744420446576656c6f706572"
  }
}

Field Descriptions

  • article_found: Indicates whether the article was located on-chain.
  • block_height: Block number where the article is included.
  • title: The title of the article.
  • content: The articleâ€™s message text (limit ~100 words).
  • publisher: The user or address that submitted the article.
  • content_hash: Hash of the article content for integrity.
  • in_pool: Whether the article is still in the mempool (unconfirmed).
  • tx_extra_nonce_hex: Raw hex-encoded data stored in the transaction extra.
  • status: Should be "OK" for a valid response.
